## Scope
Add a "Print" button to the Season Dashboard header. Tapping it builds a complete, self-contained HTML document from `PlayerSeasonData[]` and opens it in a new tab, then calls `window.print()` automatically. The print layout uses a single-page-friendly table with all players' batting and pitching stats and a team/season/date header. No external libraries.

## Out of scope
- Game-by-game breakdown
- Fielding stats in the print view
- Trend sparklines in the print view
- PDF generation libraries (Puppeteer, jsPDF, etc.)
- Cloud delivery or email

## Implementation notes
- Build a complete HTML string including `<style>` with `@media print` rules
- Open with `window.open()` + `document.write()` + `document.close()` + `window.print()` on load
- Table columns: #, Name, G, AVG, OBP, OPS, PA, AB, H, 2B, 3B, HR, R, RBI, BB, SO, SB, SLG | IP, ERA, WHIP
- Players with no pitching: leave IP/ERA/WHIP blank
- Page header: team name, season name, "Printed: {date}"
- Use Field & Form color tokens (#1B4332 primary green) in the header for brand consistency
- Format rate stats identically to `exportSeasonCsv.ts` (3-decimal, leading dot)
- Filename description (window title): `{teamName} – {seasonName} Season Report`

## Completion notes
Implemented 2026-06-11. `openPrintReport` in `printSeasonReport.ts` builds a self-contained HTML string (inline `<style>`, `@media print` rules with `print-color-adjust: exact` for header/row background), opens it in a new tab via `window.open` + `document.write` + `document.close`, then triggers `window.print()` via `window.onload`. HTML-escaping (`esc()`) applied to all user-supplied strings. Print button added to the Season Dashboard header alongside the existing CSV button; both are only rendered when `playerStats` is loaded and non-empty. A vertical divider column separates batting from pitching sections in the table.
