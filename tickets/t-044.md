## Goal
Show users their IndexedDB storage usage and nudge them to export when usage approaches 40 MB or when the browser has not granted durable storage.

## Why it matters
iOS Safari caps IndexedDB at ~50 MB and can silently evict data under storage pressure. Users need a visible signal before they lose game records, and a one-tap export path to protect their data.

## Scope
- `useStorageHealth` hook: calls `navigator.storage.estimate()` and `navigator.storage.persisted()` on mount and every 30 s
- `StorageHealthBanner` component: rendered in `SeasonDashboardScreen` when usage ≥ 40 MB or persistence is denied
- Banner includes storage usage (MB), a short explanation, and an Export CSV quick-action button

## Out of scope
- Per-table storage breakdown
- Automatic cloud backup
- A persistent status bar visible on all screens

## Implementation notes
- `navigator.storage.estimate()` returns `{ usage, quota }` in bytes
- `navigator.storage.persisted()` returns `true | false`; `false` means the OS can evict without warning
- Warning threshold: 40 MB (`usage >= 40 * 1024 * 1024`)
- Persist-denied warning: show if `persisted === false` (distinct from `null` / not checked yet)
- Use Clay Orange (`clay` design token) for the warning surface — matches Field & Form live indicators
- Banner must have a touch target ≥ 44 px for the Export CTA

## Acceptance criteria
- Banner is hidden when storage usage is < 40 MB and persistence is granted (or unknown)
- Banner appears when usage ≥ 40 MB showing exact MB used and nudge to export
- Banner appears when `navigator.storage.persisted()` returns `false` with a "storage not protected" message
- Export CSV button in the banner triggers the existing `handleExport` action
- Storage check runs on mount and refreshes every 30 s (no UI flicker between checks)
- Hook returns `null` on browsers where `navigator.storage.estimate` is unavailable (no crash)

## Test plan
