## Goal
Verify that the complete app flow works correctly on the three primary target platforms before release: iPhone Safari, Android Chrome, and iPad Safari. Identify any rendering, touch-target, layout, or PWA-install regressions before T-045 sign-off.

## Why it matters
The app is mobile-first; desktop testing in dev tools does not substitute for real device validation. iOS Safari has unique PWA constraints (no install prompt API, splash screen behavior, safe-area insets). Android Chrome has its own manifest-driven install flow. iPad requires a wider-layout sanity check even though the app is portrait-primary. T-040 and T-041 set up the PWA plumbing — T-042 confirms it works on real hardware.

## Scope
- Automated (CI-equivalent): build validation, manifest content checks, service worker content checks
- Manual checklist: install flow, offline behavior, full happy path on each of the three device targets

## Implementation notes
- Run `npm run build` and verify the dist artifacts before any device testing
- Check `dist/manifest.webmanifest` for correct icon sizes, id, orientation, start_url
- Check `dist/sw.js` for runtimeCaching entries and cleanupOutdatedCaches
- Serve dist with `npx serve dist -l 5000` for local device testing (devices must be on the same WiFi network, or use ngrok)
- iOS Safari: use "Add to Home Screen" from share sheet — there is no install banner API on iOS
- Android Chrome: look for the "Add to Home Screen" banner or use the three-dot menu
- If a bug is found during device testing, open a new ticket rather than expanding T-042 scope

## Acceptance criteria
- `npm run build` completes without errors or manifest warnings
- `dist/manifest.webmanifest` contains: `id: "/"`, `orientation: "portrait"`, PNG icons at 192×192 and 512×512 (both regular and maskable), `start_url: "/"`, `theme_color: "#1B4332"`
- `dist/sw.js` contains runtime caching for `fonts.googleapis.com` and `fonts.gstatic.com`, and `cleanupOutdatedCaches`
- On iPhone Safari: app installs via Add to Home Screen, launches with Stadium Green splash, runs all screens without layout overflow
- On Android Chrome: install banner fires or install is available via menu; app launches standalone and all screens render correctly
- On iPad: all screens are usable at 768px width; no horizontal overflow; touch targets are reachable
- Offline mode: after one online load, app loads fully with Network set to Offline (devtools or airplane mode)
- Full happy path completes on at least one device: Team → Roster → New Game → Lineup → Live Score (record 3 plate appearances) → Play Log → Game Summary → Season Dashboard

### Manual device checklist

#### All devices
- [ ] App loads at root URL without console errors
- [ ] "Add to Home Screen" / install works; icon appears on home screen
- [ ] Launching from home screen opens in standalone mode (no browser chrome)
- [ ] Stadium Green splash screen appears on iOS (apple-touch-startup-image)

- [ ] Touch targets (buttons, form fields) are large enough to tap accurately
- [ ] Back navigation works consistently across all screens

#### Happy path (one device minimum)
- [ ] Create a team + season

- [ ] Create a new game
- [ ] Build lineup (set batting order + positions for all 9)
- [ ] Navigate to live scoring; record a strikeout, a single, and a home run
- [ ] Tap undo — last play removed; tap redo/re-enter — play re-recorded

- [ ] Open game summary — box score reflects recorded plays
- [ ] Open season dashboard — player cards show correct stats

#### iPhone Safari specifics
- [ ] apple-touch-icon (180×180) shows in share sheet and on home screen
- [ ] Splash screen color matches Stadium Green (#1B4332) — no white flash
- [ ] No safe-area overflow on notch/Dynamic Island models (check bottom bar, floating buttons)
- [ ] Status bar text is legible against Stadium Green theme_color

#### Android Chrome specifics
- [ ] 192×192 and 512×512 icons are crisp; maskable icon has correct padding
- [ ] Install banner appears on second visit (or install available in three-dot menu)
- [ ] Installed app name shows "Stitch" (short_name)

#### iPad specifics
- [ ] Layout is usable at 768px viewport; no unintended landscape mode override
- [ ] Live scoring outcome grid is reachable without scrolling
