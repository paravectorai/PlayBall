## Goal
Ensure the app functions 100% offline after first load: all static assets are precached, the Inter Google Font is cached for offline use, and stale caches are cleaned up after service worker updates.

## Why it matters
The app's core promise is offline operation during live games — no network required. T-040 enabled installability; T-041 closes the gap where the Inter font (loaded from Google Fonts CDN) would fail to render offline, and where unbounded cache growth could silently accumulate across SW updates.

## Scope
- Add `runtimeCaching` entries in the Workbox config for Google Fonts:
  - `fonts.googleapis.com` (font CSS) → `StaleWhileRevalidate` so it's available offline and stays fresh when connected
  - `fonts.gstatic.com` (font binaries) → `CacheFirst` with 1-year expiry (content-addressed, immutable)
- Enable `cleanupOutdatedCaches: true` to release cache space after SW updates
- Add `navigateFallbackDenylist` to exclude browser extension URLs from the SPA fallback

## Out of scope
- Switching Inter to a locally-bundled font (T-043 can evaluate bundle tradeoffs)
- API caching (app has no network API calls — all data is IndexedDB)
- Background sync / push (V2)

## Implementation notes
- `runtimeCaching` entries use Workbox strategy strings, not class instances, in `generateSW` mode
- Font files require `cacheableResponse: { statuses: [0, 200] }` to cache opaque cross-origin responses
- `cleanupOutdatedCaches: true` is safe — it only removes caches created by older SW revisions of this app
- `navigateFallbackDenylist: [/^\/__/]` covers Chrome extension URLs (common source of noise in Lighthouse)

## Acceptance criteria
- `npm run build` completes without errors
- `dist/sw.js` contains runtimeCaching entries for `fonts.googleapis.com` and `fonts.gstatic.com`
- `dist/sw.js` contains `cleanupOutdatedCaches`
- App loads fully with fonts rendered when devtools Network is set to Offline (after at least one online load)
