# CHANGELOG.md

This file records significant project milestones, design pivots, and scope changes in human-readable form. It is not a substitute for git history — it is the narrative layer that explains what mattered and why.

## When to add an entry
Add an entry when:
- A phase or major milestone is completed
- A significant architectural decision changes the direction of the build
- A scope change is accepted or rejected
- A release gate is passed

Do not add entries for individual ticket completions — those belong in the ticket files and `TASKS.md`.

## Format

```
## YYYY-MM-DD — Short title

**Type:** Milestone / Pivot / Scope change / Gate passed

**Summary:** One or two sentences describing what happened and why it matters.

**Impact:** What this changes for future tickets or priorities.
```
