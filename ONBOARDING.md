# ONBOARDING.md

Use this checklist when joining the project mid-build or resuming in a fresh session.

## Mid-project pickup checklist
1. Read `PLAN.md` for architecture, NFRs, observability, and shared-file ownership.
2. Read `TASKS.md` for current priority, active work, blockers, and shared file ownership.
3. Read `DECISIONS.md` for prior architectural choices.
4. Read `REVIEW_CHECKLIST.md` and `CONTRIBUTING.md` before touching code or config.
5. Review all tickets marked `In Progress`, `Blocked`, or `In Review`.
6. Confirm whether the WIP limit in `TASKS.md` allows a new ticket to begin.
7. Produce a startup summary before selecting work.

## Required startup summary
Write this summary as a response before taking any other action. Then append a one-line session note to the Active Work table in `TASKS.md` so future agents can see the session context.

Include in the summary:
- Current project goal and success criteria
- Current priority and release focus
- Active tickets, owners, and files/domains they touch
- Open blockers
- Shared files or conflict areas
- Recommended next `Ready` ticket and the reason for the recommendation
